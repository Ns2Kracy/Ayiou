## 安装依赖

```toml
[dependencies]
ayiou = { git = "https://github.com/Ns2Kracy/Ayiou.git" }
anyhow = "1.0"
tokio = { version = "1", features = ["full"] }
async-trait = "0.1"
```

默认安装只包含 core runtime、插件 API、宏和共享 trait。使用内置 adapter 时需要显式启用 feature：

```toml
ayiou = { git = "https://github.com/Ns2Kracy/Ayiou.git", features = ["adapter-onebot-v11"] }
```

Console 本地调试则启用：

```toml
ayiou = { git = "https://github.com/Ns2Kracy/Ayiou.git", features = ["adapter-console"] }
```

需要内嵌控制面管理 UI 时启用：

```toml
ayiou = { git = "https://github.com/Ns2Kracy/Ayiou.git", features = ["adapter-console", "embedded-webui"] }
```

## 核心概念

- `Bot<A>`：单 Bot 运行时 builder
- `Adapter / ProtocolAdapter`：把平台事件归一化到内核
- `RuntimePlugin`：处理消息、命令、生命周期、配置和主动发送
- `Context`：默认通用上下文

## 示例 1：OneBot v11

需要启用 `adapter-onebot-v11`。

```rust
use ayiou::{plugin, Context, OneBotV11Bot};

#[derive(Default)]
struct EchoPlugin;

#[plugin(name = "echo", prefix = "/")]
impl EchoPlugin {
    async fn echo(&self, ctx: &Context) -> anyhow::Result<()> {
        ctx.reply_text(format!("echo: {}", ctx.text())).await?;
        Ok(())
    }
}

#[tokio::main]
async fn main() {
    let bot = OneBotV11Bot::ws("ws://127.0.0.1:8080").command_prefixes(["/", "!", "."]);

    bot.run().await;
}
```

## 示例 2：Console

需要启用 `adapter-console`。

```rust
use ayiou::{plugin, ConsoleBot, Context};

#[derive(Default)]
struct HelpPlugin;

#[plugin(name = "help", prefix = "/")]
impl HelpPlugin {
    async fn help(&self, ctx: &Context) -> anyhow::Result<()> {
        ctx.reply_text("Commands: /help").await
    }
}

#[tokio::main]
async fn main() {
    let bot = ConsoleBot::console();
    bot.run().await;
}
```

## 平台扩展

默认推荐直接使用 `Context`。当你需要平台原生 API 时，再通过扩展访问平台上下文。

Console 示例：

```rust
if let Some(console) = ctx.extension::<ConsoleCtx>() {
    let raw_line = console.line();
}
```

OneBot 示例：

```rust
if let Some(onebot) = ctx.extension::<Ctx>() {
    let login = onebot.get_login_info_data().await?;
}
```

## 高级插件

默认入口只推荐 `#[plugin]`。如果需要非命令 handler、能力声明、配置生命周期或复杂会话控制，手写 `RuntimePlugin` 并从 `ayiou::core::plugin` 引入运行时类型。

## 控制面

控制面是可选能力。`bind` 可选，默认 `127.0.0.1:32187`；`token` 必填。

```rust
use ayiou::{ConsoleBot, ControlPlaneOptions};

#[tokio::main]
async fn main() {
    ConsoleBot::console()
        .control_plane(ControlPlaneOptions::new().token("change-me"))
        .run()
        .await;
}
```

启用 `embedded-webui` 后打开 `http://127.0.0.1:32187/`。当前 Rust 插件支持 enable/disable/start/stop；代码热加载会由后续 Rhai/WASM backend 接入同一套 reload 入口。

## Runtime Services

Plugins should not call other plugin instances directly. Register host-provided services on `Bot`, then request them from `RuntimePluginServices` during plugin initialization:

```rust
let bot = Bot::new(adapter)
    .with_service(AclService::new())
    .with_plugin(AdminPlugin);
```

```rust
fn manifest(&self) -> RuntimePluginManifest {
    RuntimePluginManifest::new("admin")
        .require_service::<AclService>()
}
```

```rust
async fn init(&mut self, services: RuntimePluginServices<Context>) -> anyhow::Result<()> {
    self.acl = Some(services.require_service::<AclService>()?);
    Ok(())
}
```

Required service declarations are preflighted before any plugin `init()` runs, so a missing dependency fails startup without partially initializing the plugin set.

For diagnostics or adaptive behavior, plugins can inspect registered service metadata through `services.service_descriptors()`. Descriptors include the service type key, `RuntimeService::name()`, and `RuntimeService::version()`.

## 常用 Context API

- `ctx.text() -> String`
- `ctx.user_id() -> String`
- `ctx.group_id() -> Option<String>`
- `ctx.reply_text("...")`
- `ctx.extension::<ConsoleCtx>()` / `ctx.extension::<Ctx>()`
