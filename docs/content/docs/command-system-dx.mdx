title: 命令系统与 DX
description: 使用 #[plugin] 构建可维护的命令型插件
---

`#[plugin]` 适合“多命令、强签名、自动参数解析”的插件。impl 里的 async 方法会按方法名注册成命令。

## 快速示例

这个示例使用 `OneBotV11Bot`，依赖需要启用 `adapter-onebot-v11` feature。

```rust
use ayiou::{plugin, Context, OneBotV11Bot};

#[derive(Default)]
struct AdminPlugin;

#[plugin(name = "admin", prefix = "/", prefix = "!")]
impl AdminPlugin {
    async fn ban(&self, ctx: &Context, user_id: i64, minutes: Option<u32>) -> anyhow::Result<()> {
        let minutes = minutes.unwrap_or(10);
        ctx.reply_text(format!("ban {} for {} minutes", user_id, minutes)).await?;
        Ok(())
    }

    async fn say(&self, ctx: &Context, content: String) -> anyhow::Result<()> {
        ctx.reply_text(content).await?;
        Ok(())
    }
}

#[tokio::main]
async fn main() {
    let bot = OneBotV11Bot::ws("ws://127.0.0.1:8080").command_prefixes(["/", "!", "."]);

    bot.run().await;
}
```

## 参数解析规则

命令方法签名要求：

1. 必须是 `async fn`
2. 第一个参数必须是 `&self`
3. 第二个参数默认推荐写成 `&Context`
4. 后续参数自动从命令参数解析

支持类型：

- 任意 `FromStr`
- `Option<T>`
- 尾部 `String`
- 尾部 `Vec<String>`

## 命令匹配

- `async fn kick(...)` 会注册 `kick`
- 运行时会依次尝试：
- 全局前缀：`Bot::command_prefixes(...)`
- 插件前缀：`#[plugin(prefix = "...")]`

默认推荐把第二个参数写成 `&Context`，这样命令解析、回复和跨平台插件体验最稳定。

如果确实需要平台原生 API，再通过：

- `ctx.extension::<Ctx>()`
- `#[plugin(context = "...")]`

访问平台上下文。

## 输入示例

```text
/say "hello world" foo\ bar
```

解析结果：

```text
["hello world", "foo bar"]
```
