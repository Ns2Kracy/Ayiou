`#[derive(Plugin)]` 会为你的结构体自动实现 `ayiou::core::plugin_system::RuntimePlugin`。

默认推荐直接面向通用 `Context` 编写插件；只有在你明确依赖平台专属 API 时，才切到平台上下文。

## 基本用法

```rust
use ayiou::prelude::*;

#[derive(Plugin)]
#[plugin(name = "my_plugin", command = "hello", prefix = "/")]
struct MyPlugin;

impl MyPlugin {
    async fn execute(&self, ctx: &Context) -> anyhow::Result<()> {
        ctx.reply_text("hello").await?;
        Ok(())
    }
}
```

## 可用属性

| 属性 | 说明 |
|---|---|
| `name` | 插件名，默认是结构体名的小写 |
| `description` | 插件描述 |
| `version` | 插件版本，默认 `0.1.0` |
| `command` | 命令名；如果未提供且不是 `regex/cron` 插件，则默认使用 `name` |
| `prefix` | 插件级命令前缀，可重复声明 |
| `regex` | 消息匹配正则 |
| `cron` | 调度表达式 |
| `context` | 显式指定上下文类型 |
| `start` | 自定义启动方法，签名应为 `async fn xxx(&mut self, host: PluginHost<Context>) -> Result<()>` |
| `handler` | 自定义处理方法，签名应为 `async fn xxx(&self, ctx: &Context) -> Result<bool>` |

## 平台扩展

默认上下文是 `Context`。通用插件优先通过 `RuntimePluginManifest` 声明能力需求；当你确实需要平台专属 API 时，再通过 `ctx.extension::<...>()` 获取。

```rust
use ayiou::adapter::onebot::v11::model::Message;
use ayiou::prelude::*;

#[derive(Plugin)]
#[plugin(name = "whoami", command = "whoami")]
struct WhoAmI;

impl WhoAmI {
    async fn execute(&self, ctx: &Context) -> anyhow::Result<()> {
        if let Some(onebot) = ctx.extension::<Ctx>() {
            let login = onebot.get_login_info_data().await?;
            onebot
                .reply(Message::String(format!(
                    "bot={} user={}",
                    login.nickname,
                    onebot.user_id()
                )))
                .await?;
        }

        Ok(())
    }
}
```

如果平台上下文本身就是你的主入口，也仍然可以使用：

```rust
#[derive(Plugin)]
#[plugin(context = "ayiou::prelude::Ctx")]
struct OneBotOnlyPlugin;
```

## 高级用法

```rust
use std::sync::OnceLock;

use ayiou::core::plugin_host::PluginHost;
use ayiou::prelude::*;

#[derive(Plugin)]
#[plugin(
    name = "stateful",
    description = "stateful plugin",
    start = "start_plugin",
    handler = "handle_command"
)]
struct StatefulPlugin {
    store: OnceLock<std::sync::Arc<dyn ayiou::core::storage::Store>>,
}

impl StatefulPlugin {
    async fn start_plugin(&mut self, host: PluginHost<Context>) -> anyhow::Result<()> {
        let _ = self.store.get_or_init(|| host.store());
        Ok(())
    }

    async fn handle_command(&self, ctx: &Context) -> anyhow::Result<bool> {
        ctx.reply_text(format!("received: {}", ctx.text())).await?;
        Ok(true)
    }
}
```

## 什么时候用 #[bot_plugin]

`#[derive(Plugin)]` 更适合：

- 单入口命令插件
- 通用业务插件
- 需要 `start` / `handler` 的状态型插件

`#[bot_plugin] + #[command]` 更适合：

- 多命令插件
- 需要自动参数解析的命令型插件
- 命令签名较多、希望方法级分发更清晰的场景
