# Ayiou v1 目标

Ayiou v1 不再把自己定义成“某个平台 SDK”，而是一个面向 IM Bot 的通用框架内核。

v1 的第一成功标准是工程成熟度，而不是平台数量。首批一等适配器只有 `OneBot + Console`，但所有公共抽象都要避免把 OneBot 私有结构直接泄漏到内核 API。

## v1 交付边界

包含：

- 平台无关的消息型插件开发体验
- 面向多 Bot 的统一运行时和程序化管理 API
- 配置、会话、主动消息、可观测等核心运行时服务
- 静态注册插件类型，运行时按配置启用已知插件

不包含：

- 内建完整权限系统
- CLI / Web UI
- 完全动态插件安装与 marketplace
- 沙箱执行、多租户、审计系统

## 架构分层

v1 统一采用三层模型：

- `Kernel`：统一事件模型、插件契约、调度、存储、会话、可观测、错误模型、执行管线
- `Adapter`：平台入站事件归一化、平台能力映射、主动发送实现
- `Supervisor`：多 Bot 注册、生命周期管理、插件启停、配置应用、状态查询

其中 `Bot<A>` 在 v1 中只保留“单 bot runtime builder”的角色；多 Bot 管理入口提升为 `Supervisor` / `BotRegistry`。

## 核心模型

v1 引入平台无关的统一消息模型，作为插件默认面对的公共接口：

- `BotId`
- `PlatformId`
- `ChannelRef`
- `UserRef`
- `EventEnvelope`
- `MessageEvent`
- `OutboundMessage`
- `MessageSegment`
- `CommandInvocation`

v1 的通用范围只覆盖消息优先能力：

- 收消息
- 命令解析
- 用户 / 频道标识
- 基础消息段与附件
- 主动发送

`edit/delete/reaction/member-change` 等能力不进入 v1 通用模型，只能通过适配器扩展接口暴露。

## 插件与运行时

插件模型采用“双入口、通用优先”：

- 通用业务插件：基于统一消息、命令、会话 API 编写
- 平台扩展插件：显式声明依赖平台扩展能力

运行时能力以实例为中心，统一为：

- `init`
- `start`
- `stop`
- `handle_event`
- `apply_config`
- `health` / `status`

配置语义采用显式应用：

- 写配置只更新 `desired config version`
- 调用 `apply_config` / `reload_plugin` 后才推进 `applied config version`
- 运行时必须记录两者差异

`SessionStore` 作为内核正式服务，最小 key 维度固定为：

- `bot`
- `platform`
- `plugin`
- `user`
- `channel`

## 管理 API

v1 管理面只定义 Rust trait / API，不把 HTTP / gRPC 规定为标准面。

最小管理对象：

- `BotManager`
- `PluginManager`
- `ConfigManager`

最小操作集合：

- 注册 / 删除 bot 定义
- 启动 / 停止 / restart bot
- 查询 bot 状态与适配器信息
- 列出 bot 插件实例
- 启用 / 禁用 / reload 插件
- 读写配置
- 对指定插件实例应用配置

Supervisor 内部固定模型：

- 全局 `PluginCatalog`
- 多 bot `BotRegistry`
- 每 bot 独立 `BotRuntime`
- 每 bot 独立插件实例集合与配置快照

## 健壮性要求

- 插件错误必须向运行时显式上浮并记录，不允许静默吞错
- 单 bot 事件执行改为有界并发模型，不能继续无界 `spawn`
- 停止 / 重载时需要可取消未完成任务
- 可观测性默认内建到运行时
- 适配器需要暴露能力声明与错误分类

## 本次仓库内首批落地

这一轮实现先聚焦于“以后最难改”的公共骨架：

- 新增平台无关核心消息模型
- `MessageSender` 重构为平台无关 outbound 接口
- 插件实例状态扩展为 `enabled + desired/applied config version + lifecycle + last error`
- 新增 `PluginCatalog` / `Supervisor` 管理骨架
- `Bot` 运行时改为有界队列 + worker 模式
- 宏不再硬编码 `/ ! .` 作为唯一命令前缀来源

## 非目标与后续

下列内容在本轮实现中只预留接口，不追求一次完成：

- 通用 `Context` 全量替换现有平台专属 `Ctx`
- OneBot / Console 全量 contract tests
- 权限扩展点与拦截器
- 完整插件 reload 语义
- SeaORM 存储层的正式化或移除

后续迭代应继续把现有适配器与插件示例逐步迁移到统一 `Context` 路径，并补齐多 Bot 集成示例与升级文档。
