`ayiou/examples/kitchen_sink.rs` 是一个可编译、可运行的宏插件 smoke flow。

它不连接真实 OneBot，也不阻塞 Console 输入，而是用内存里的 adapter 投递几条消息事件，验证 `#[plugin]` 插件会被 `Bot::run()` 自动发现并执行。启用 `control-plane` / `embedded-webui` feature 时，它还会演示 Bot builder 如何挂载控制面。

## 运行方式

在仓库根目录执行：

```bash
cargo run -p ayiou --example kitchen_sink
```

启用控制面 API：

```bash
cargo run -p ayiou --features control-plane --example kitchen_sink
```

启用内嵌管理 UI：

```bash
cargo run -p ayiou --features embedded-webui --example kitchen_sink
```

也可以只检查 example 是否能编译：

```bash
cargo test -p ayiou --example kitchen_sink
```

成功运行时会看到类似输出：

```text
control plane: http://127.0.0.1:32187/ token=kitchen-sink-token
echo: hello from macro
add: 42
plugin macro handled: /hello
kitchen sink example completed
```

默认不启用控制面 feature 时，不会打印第一行。

## 覆盖了什么

### 宏生成 RuntimePlugin

示例只使用一个默认宏入口：

- `#[plugin]`：声明插件元信息、统一 `Context` 和自动发现入口
- async 方法：按方法名注册成命令，并按签名自动解析参数

宏插件默认会自动注册。示例的 Bot 入口没有逐个 `.with_plugin(HelloPlugin)` / `.with_plugin(ToolsPlugin)`，`Bot::run()` 会按统一 `Context` 自动加载它们。

```rust
#[derive(Default)]
struct HelloPlugin;

#[plugin(name = "hello", prefix = "/")]
impl HelloPlugin {
    async fn hello(&self, ctx: &ayiou::Context) -> anyhow::Result<()> {
        println!("plugin macro handled: {}", ctx.text());
        Ok(())
    }
}
```

### 命令参数解析

`ToolsPlugin` 展示了尾部 `String` 和 typed 参数解析：

```rust
#[derive(Default)]
struct ToolsPlugin;

#[plugin(name = "tools", prefix = "/")]
impl ToolsPlugin {
    async fn echo(&self, _ctx: &ayiou::Context, content: String) -> anyhow::Result<()> {
        println!("echo: {content}");
        Ok(())
    }

    async fn add(&self, _ctx: &ayiou::Context, left: i64, right: i64) -> anyhow::Result<()> {
        println!("add: {}", left + right);
        Ok(())
    }
}
```

### Bot builder

```rust
let bot = Bot::new(DemoAdapter)
    .workers(1)
    .queue_capacity(8)
    .command_prefixes(["/"]);

#[cfg(feature = "control-plane")]
let bot = bot.control_plane(
    ControlPlaneOptions::new()
        .bind("127.0.0.1:32187")
        .token("kitchen-sink-token"),
);

bot.run().await;
```

这说明 Ayiou 的公开主路径是一个 Bot runtime 加宏插件自动发现。控制面也是 Bot builder 的可选能力：host/bind 可配置，token 必填。手写插件仍然属于高级扩展路径，但默认示例不再展示它。

## 什么时候参考它

- 你想确认 `#[plugin]` 应该怎么使用
- 你想确认宏插件自动发现是否生效
- 你要理解 `Bot` builder 如何承载默认插件运行路径
- 你要确认控制面如何挂到 Bot builder 上
