`ayiou/examples/kitchen_sink.rs` 是一个可编译、可运行的全特性示例。

它不连接真实 OneBot，也不阻塞 Console 输入，而是用内存里的 adapter 和 outbound sender 跑一遍 smoke flow。这样你可以安全地用它检查内核 API 是否能组合起来工作。

## 运行方式

在仓库根目录执行：

```bash
cargo run -p ayiou --example kitchen_sink
```

也可以只检查 example 是否能编译：

```bash
cargo test -p ayiou --example kitchen_sink
```

成功运行时会看到类似输出：

```text
capability negotiation: Degraded { missing_optional: [Reaction] }
bonjour secure command args=admin open sesame
echo: hello from macro
plugin macro handled: /hello
kitchen sink example completed
```

## Runtime Services

This example registers a host-provided `GreetingService` on `Bot`, then requests it from `RuntimePluginServices` during `KitchenPlugin::init()`.

Plugins should not call other plugin instances directly. Register host-provided services on `Bot`, then request them from `RuntimePluginServices` during plugin initialization:

```rust
let bot = Bot::new(adapter)
    .with_service(AclService::new())
    .with_plugin(AdminPlugin);
```

```rust
async fn init(&mut self, services: RuntimePluginServices<Context>) -> anyhow::Result<()> {
    self.acl = Some(services.require_service::<AclService>()?);
    Ok(())
}
```

## 覆盖了什么

### 宏生成 RuntimePlugin

示例里使用同一套默认宏入口：

- `#[plugin]`：声明插件元信息和默认上下文
- `#[command]`：声明命令、别名和参数解析入口

```rust
#[derive(Default)]
struct HelloPlugin;

#[plugin(name = "hello", prefix = "/", context = "DemoCtx")]
impl HelloPlugin {
    #[command(name = "hello")]
    async fn execute(&self, ctx: &DemoCtx) -> anyhow::Result<()> {
        println!("plugin macro handled: {}", ctx.text());
        Ok(())
    }
}

#[derive(Default)]
struct ToolsPlugin;

#[plugin(name = "tools", prefix = "/", context = "DemoCtx")]
impl ToolsPlugin {
    #[command(name = "echo", alias = "say")]
    async fn echo(&self, _ctx: &DemoCtx, content: String) -> anyhow::Result<()> {
        println!("echo: {content}");
        Ok(())
    }
}
```

这个宏入口生成的是 `RuntimePlugin` 实现，而不是旧的消息插件接口。

### 手写 RuntimePlugin

`KitchenPlugin` 展示了手写运行时插件的完整形态：

- `meta()`：插件元信息
- `manifest()`：必需/可选能力声明
- `declared_handlers()`：命令、正则、权限、优先级和 block
- `init()` / `start()` / `stop()`：生命周期
- `apply_config()`：配置 dry-run 与实际应用
- `handle_with_invocation()`：命令处理入口
- `handle()`：非命令 handler 入口
- `health()`：健康状态

```rust
fn manifest(&self) -> RuntimePluginManifest {
    RuntimePluginManifest::new(self.kind())
        .require_capability(Capability::ProactiveSend)
        .optional_capability(Capability::Reaction)
}

fn declared_handlers(&self) -> Vec<HandlerDecl> {
    vec![
        HandlerDecl::message_commands(["secure"], ["/"])
            .require_permission(Permission::user("admin"))
            .priority(5)
            .block(true),
        HandlerDecl::message_regex(["(?i)kitchen\\s+sink"])
            .require_permission(Permission::PlatformCapability(Capability::ProactiveSend))
            .priority(20),
    ]
}
```

### Handler 管线

示例会依次触发：

- `/secure admin open sesame`：命令 handler，带权限、优先级和 block
- `please show the kitchen sink`：正则 handler，要求 `ProactiveSend` 能力
- `/say hello from macro`：`#[plugin] + #[command]` 生成的命令 handler
- `/hello`：同一套宏生成的另一个命令 handler

这对应 v0.2 的统一 handler 管线：adapter 产出事件，runtime 做粗筛、rule、permission 和 handler 执行。

### 配置生命周期

示例同时展示 dry-run 与实际 apply：

```rust
self.apply_config(ConfigUpdate::dry_run(1, "bonjour")).await?;
self.apply_config(ConfigUpdate::new(2, "bonjour")).await?;
```

运行时状态会记录：

- dry-run：进入 `validated`，不更新 applied version
- apply：进入 `applied`，更新 applied version

### Bot builder

```rust
Bot::new(adapter)
    .with_service(GreetingService {
        greeting: "host service".to_string(),
    })
    .with_plugin(HelloPlugin)
    .with_plugin(ToolsPlugin)
    .with_plugin_as("kitchen-main", KitchenPlugin::new())
    .workers(1)
    .queue_capacity(8)
    .command_prefixes(["/"])
    .run()
    .await;
```

这说明 Ayiou 的公开主路径是一个 Bot runtime。多个消息渠道应该作为 adapter/source 接入同一个 runtime，回复时根据消息来源回到原渠道。

- 你想看配置、能力协商和 handler 管线如何一起工作
